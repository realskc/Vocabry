# ADR-0006：第一版只提供 API 管理边界

- 状态：已接受、已实现

## 背景

本地 HTTP API 已覆盖导入、Card 管理、preview session、配对和同步状态。同期建设完整 CLI 还需要命令结构、交互编辑、输出格式和错误呈现，但本质上只是 API 的另一个客户端。

早期产品主要由维护者通过 Coding Agent、脚本或 API 工具操作，独立 CLI 的人类体验尚未被需求证明。

## 决策

第一版不提供完整管理 CLI。API 是正式管理边界，未来 CLI 或 GUI 都应作为可替换客户端使用它。

`vocabd` 只提供 `start` 和简化的 `status` 生命周期命令，不承载 Card 管理。旧设计中提到的 install/stop 尚未实现，也不属于这项决策已经交付的部分。

## 代价

- 不使用 Agent 或 API 工具时，人类管理卡片不方便。
- 调用者需要处理 token、JSON、幂等 key 和 revision conflict。
- API 必须覆盖所有正式管理能力。

## 重新评估条件

当非开发者需要直接管理卡片、多个高频操作反复依赖手写请求，或脚本开始重复实现认证与错误处理时，可基于稳定 API 增加薄 CLI 或 GUI。业务规则与 SQLite 访问仍不进入客户端。
