# ADR-0002：`vocabd` 是唯一业务数据库所有者

- 状态：已接受、已实现

## 背景

管理客户端、预览器和 Add-on 都需要卡片数据。如果每个组件直接操作 SQLite，业务规则、迁移、并发控制、历史和 outbox 事务会分散到多个进程。

## 决策

只有 `vocabd` 打开业务 SQLite。其他组件通过本地 HTTP/WebSocket API 或离线导入协议交互。数据库内部维护脚本也应复用明确的服务/数据库方法，不能成为长期旁路。

## 代价

- API 客户端必须确保服务可用并持有凭据。
- 单机产品仍需处理服务生命周期、认证和协议兼容。
- 简单查询多一次进程间通信。

## 重新评估条件

若未来改用天然支持多客户端事务和权限的存储服务，可以重新划分所有权；多个进程直接共享当前 SQLite 不是默认退路。
