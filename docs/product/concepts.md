# 核心概念

本文解释使用 Vocabry 时会遇到的名词。它面向不熟悉 Anki 或软件实现的读者，不描述数据库表和接口字段。

## Anki 是什么

Anki 是一个使用间隔重复帮助记忆的复习软件。用户看到问题面，尝试回忆答案，再翻到答案面并评价难度；Anki 根据这些反馈决定下一次出现时间。

Vocabry 不参与这个排程。它只向 Anki 提供复习内容。

## Note、Card 与 Note Type

Anki 的几个术语容易混淆：

- **Note（笔记）**是一组内容字段。例如一个词汇 Note 可以包含单词、正面 HTML 和背面 HTML。
- **Note Type（笔记类型）**定义 Note 有哪些字段，以及怎样从字段生成复习卡片。
- **Card（Anki 卡片）**是实际进入复习队列的问答方向。一个 Note 可以生成一张或多张 Card。

例如，同一个法语单词 Note 可以生成“法语 → 中文”和“中文 → 法语”两张 Anki Card。Vocabry 第一版使用专用的 `Vocabry` Note Type，每个受管 Note 只生成一张 Anki Card，字段为：

| 字段 | 用途 |
|---|---|
| `ExternalCardId` | 关联 Vocabry 中的稳定身份 |
| `Front` | 复习时的问题面 HTML |
| `Back` | 翻面后的答案面 HTML |

## Vocabry Card

Vocabry 也使用 “Card” 一词，但它表示卡片库中的内容实体，不是 Anki 排程中的 Card。每个 Vocabry Card 包含稳定 ID、卡型、结构化词汇字段、当前正反面 HTML、版本历史、可选导入来源和删除状态。

第一版约定一张 Vocabry Card 对应一个受管 Anki Note，再由这个 Note 生成一张 Anki Card。

## 两种卡型

两种卡型使用相同字段，但问题面不同：

| 卡型 | 正面 | 背面 |
|---|---|---|
| `standard_definition` | 单词和例句 | 单词、音标、释义、例句、备注 |
| `single_definition_word` | 单词 | 单词、音标、释义、例句、备注 |

`word` 和 `definition` 必填，其余字段可以为空。所有字段都被当作纯文本；输入 `<b>word</b>` 不会获得加粗效果，而会按原文字样安全显示。换行会被转换成 HTML 换行。

## 结构化字段与当前 HTML

Vocabry 同时保存两种表示：结构化字段便于生成器和管理工具理解内容；当前 HTML 是实际交给 Anki 和预览器的卡面。

通常 HTML 由 Vocabry 根据结构化字段生成。但用户可以在 Anki 中直接修改 `Front` 或 `Back`，此时手工 HTML 会成为当前版本，结构化字段不会被反向猜测或修改。

后续行为遵循一个可见规则：

1. Anki 手工修改后，手工 HTML 保留。
2. 仅仅升级 Vocabry 的渲染器，不应覆盖手工 HTML。
3. 用户或工具再次修改结构化字段时，Vocabry 会重新生成 HTML，先前的手工版本进入历史、不再作为当前卡面。

这是有意的取舍：系统不会假装能从任意 HTML 准确还原单词、释义等字段。

## Revision（版本号）

每张卡从 revision 1 开始。有效的内容修改、Anki HTML 回写、重新渲染或首次删除都会使 revision 增加；提交完全相同的内容不会制造新版本。

编辑时，客户端必须说明自己看到的 revision。若卡片已经被其他操作修改，旧请求会被拒绝，而不是悄悄覆盖新内容。revision 因此既是历史序号，也是并发保护。

当前版本历史可以查询，但产品还没有“恢复到旧版本”的操作。

## 删除与缺失

Vocabry 使用软删除：卡片仍保留在数据库和历史中，但默认列表不再显示，并向 Anki 发送删除事件。

“明确删除”和“找不到 Note”在产品含义上并不完全相同。理想同步设计应区分用户明确删除与映射漂移；当前 Add-on 通过操作后扫描判断 Note 消失，并将其作为删除回传，这部分仍需要真实 Anki 环境验证。

## 导入任务

导入任务（job）是一批必须一起成功或一起失败的卡片文件。它有稳定的 job ID、生成器信息、卡片列表和最终结果。生成器不需要直接连接数据库，用户也能保留可检查的输入与错误报告。

重复使用已经处理过的 job ID 会失败；重新生成应使用新的 ID。

## 配对、token 与预览 session

- **管理 token**在服务首次启动时生成，拥有管理能力，保存在数据目录的 `admin.token`。
- **配对码**由管理客户端创建，5 分钟内只能使用一次。Anki Add-on 用它换取自己的长期 token。
- **预览 session**只允许查看指定卡片，10 分钟后过期。浏览器不需要也不应获得管理 token。

这些凭据都只用于当前电脑上的本地服务；“仅本机”仍不等于“无需认证”，因为网页或同机其他程序也可能访问 localhost。
