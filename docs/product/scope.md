# Vocabry 产品范围

## 目标

建立一个与复习软件解耦的本地卡片库。外部生成器通过离线文件协议提交卡片，主应用负责校验、存储、管理、生成 HTML，并与 Anki 双向同步。

## 第一版用户路径

1. 外部生成器把一个完整 job 原子投递到 inbox。
2. 用户启动 `vocabd`，随后运行 `vocab ingest`，或通过裸 `vocab` / `vocab sync` 隐式触发导入。
3. 合法 job 整批写入；非法 job 整批失败，并产生机器可读报告。
4. Anki Add-on 与 `vocabd` 配对，将本地卡片同步为 Anki Note。
5. 用户在 Anki 中修改 HTML 或删除 Note，Add-on 将变化回写本地。
6. 用户可通过 CLI 查询、编辑和删除卡片，并通过浏览器预览当前 HTML。

## 第一版范围

- Windows 10/11；核心代码尽量保持跨平台。
- 单用户、单机、单 SQLite 数据库、单 Anki Profile。
- 两种内置卡型，字段均为纯文本。
- 外部生成器始终新增卡片，不去重、不按指纹更新。
- 完整版本历史，但不提供显式恢复接口。
- 本地 HTTP API、认证 WebSocket、独立 CLI、Anki Add-on、浏览器预览页。
- Anki 在线时实时通知；离线重连后增量对账。

## 非目标

- 在本应用中实现复习算法或复习界面。
- 内置 LLM、词典或网页抓取生成器。
- Markdown、音频、图片或任意用户自定义卡型。
- 多设备、多用户、多 Anki Profile 或分布式冲突解决。
- 自动语义去重、自动修正与卡片合并。
- 第一版提供版本恢复和跨平台后台服务安装体验。

## 验收标准

- 合法 job 可原子导入并生成可用 HTML；任一行错误时数据库零新增。
- 错误报告足以让程序定位到文件、行、字段和错误代码。
- 本地新增、编辑、删除最终准确反映到 Anki。
- Anki 对受管 Note 的 HTML 修改和删除最终准确反映到本地。
- 中断连接、重复消息和进程重启不会造成漏同步或重复创建。
- 未携带有效 token 的请求无法读取或修改数据。
