# Vocabry 系统不变量

> 文档角色：集中保存跨组件、不能被局部重构悄悄破坏的规则。编号用于组件、协议、ADR、测试和任务上下文相互引用。

当前仓库仍处于设计阶段，因此“验证证据”一栏描述计划中的落点；实现后必须替换为真实 schema、测试或监控入口。

## 数据所有权与事务

### `DATA-001` 单一数据库所有者

只有 `vocabd` 可以打开和写入业务 SQLite。API 客户端、预览器、Add-on 和生成器必须通过已定义边界交互。

计划证据：包边界测试；客户端依赖检查；数据库文件权限与进程集成测试。

### `IMPORT-001` Job 全有或全无

一个离线导入 job 的 Card、HTML、来源和初始 revision 必须在同一个数据库事务中提交。任何 manifest、编码、JSONL 或字段错误都导致零新增。

计划证据：合法/非法 golden files；事务回滚集成测试；提交前故障注入。

### `CHANGE-001` 当前态、历史和事件一致提交

一次 Card 编辑或删除产生的当前态、不可变 revision 和 outbox event 必须位于同一事务。系统不能出现状态已经改变但对应同步事件永久丢失的情况。

计划证据：数据库约束；事务集成测试；提交前后故障注入。

## Card 与 HTML

### `CARD-001` 稳定身份与单调版本

`card_id` 创建后永久稳定。每次有效状态变化都追加 revision，且 revision 对同一 Card 单调递增。

计划证据：唯一约束；状态机单元测试；并发写入测试。

### `CARD-002` 删除可同步且可审计

明确的本地或 Anki 删除都转换为软删除和新 revision。墓碑至少保留到当前 Anki 客户端确认删除，历史在第一版不做物理清理；对账时仅发现 Note 缺失不构成明确删除。

计划证据：删除状态机测试；断线删除后的重连对账测试。

### `HTML-001` 自动渲染只依赖结构化字段

内置 renderer 必须是确定性的，并只接受对应 schema 的结构化纯文本字段。结构化字段变化会重新生成当前 HTML。

计划证据：schema；renderer golden tests；相同输入产生相同输出的测试。

### `HTML-002` 手工 HTML 不被升级静默覆盖

Anki 回写可以使 `html_origin=anki_manual` 并成为当前 HTML。单纯升级 renderer 不能覆盖它；只有之后的结构化字段修改才重新渲染并取代手工 HTML。

计划证据：renderer/Anki/manual/edit 状态机测试。

## API 与同步

### `API-001` 过期写入不得静默覆盖

修改或删除既有 Card 的请求必须携带 `expected_revision`。基于旧 revision 的请求返回 `409 revision_conflict`，客户端重新读取后再决定是否重试。

计划证据：OpenAPI 契约测试；两个客户端并发写入测试。

### `SYNC-001` Card 与受管 Note 一一映射

第一版中，一张活动 Card 至多对应一个受管 Anki Note；`ExternalCardId` 是 Anki Note 丢失本地 `note_id` 映射后的重新识别依据。

计划证据：数据库唯一约束；身份漂移对账测试。

### `SYNC-002` 事件消费幂等

同一个 `(card_id, revision)` 可以重复投递和消费，不得因此重复创建 Note 或重复产生业务变更。

计划证据：重复事件和 ack 丢失集成测试。

### `SYNC-003` 成功后才推进游标

客户端游标只能在对应事件成功应用并持久化结果后推进。失败、进程退出或 ack 丢失必须允许安全重放。

计划证据：应用前后、ack 前后故障注入。

### `SYNC-004` 较旧事件不能覆盖更新的 Anki 用户操作

Add-on 重连时必须先保全离线期间观察到的用户修改，再进行服务事件对账。系统不使用跨进程墙上时钟决定胜负，而以 `vocabd` 接受并提交变更的顺序为准。

计划证据：离线编辑、乱序事件和重连集成测试。

## 安全

### `SEC-001` 业务边界默认不公开

API 只绑定 loopback。除最小健康检查和一次性配对入口外，HTTP 与 WebSocket 都必须认证；服务不启用 CORS。

计划证据：未认证访问、Host/Origin 和 WebSocket 鉴权测试。

### `SEC-002` 浏览器不持有管理能力

预览页面只能使用短时、限定用途的 session，不能获得长期管理 token。Anki 手工 HTML 必须在严格 sandbox iframe 中显示。

计划证据：浏览器安全测试；session 权限和过期测试；iframe sandbox 断言。

### `SEC-003` 外部文本和路径不受信任

结构化纯文本在渲染前统一 HTML 转义。job ID、载荷文件名和解析后路径必须经过允许列表与目录边界检查，并设置资源上限。

计划证据：路径穿越、HTML 注入和资源上限测试。

## 维护规则

- 只有跨越实现细节、长期成立并会影响修改正确性的规则才进入本文件。
- 新增或改变不变量时，同时更新引用它的组件、协议、ADR 和验证计划。
- 实现完成后，每条不变量至少链接一个可执行证据；只有文字说明不能证明约束成立。
- 若业务决定允许违反某条不变量，先用 ADR 记录原因和迁移方式，再修改或废弃其编号；不要直接复用编号表达新含义。
