# 开发规范

本文只记录 Vocabry 容易被遗漏、且无法从一般工程常识直接推出的约束。

## Bug 回归证据

修复 Bug 时，回归测试必须在修复前确实捕获该 Bug，并确认失败来自目标行为，而非测试错误或环境故障。只有修复后的绿灯、没有修复前的有效红灯，不足以证明测试有用。

若旧环境不可恢复或缺陷依赖不可控外部状态，可以采用等价证据，但应说明为什么无法直接获得红灯。纯文档、机械改名等不改变运行行为的修改不适用这一要求。

## 当前事实文档

代码改变用户行为、产品能力、协议、数据模型、配置、运行方式、验证边界或已知限制时，应同步对应的当前事实文档：

- 产品事实：`docs/product/`
- 技术语义：`docs/design/`
- 已实现、已验证与风险：`docs/STATUS.md`
- 仓库入口和安装方式：根 `README.md`、`docs/README.md`

`docs/archive/` 只保存历史，不能代替当前事实文档。纯内部重构若不改变上述事实，无需为了留痕而修改文档。

## 测试范围

项目当前把真实 DeepSeek 调用算作普通完整测试。只运行离线子集时，应明确区分离线结果与完整结果。
