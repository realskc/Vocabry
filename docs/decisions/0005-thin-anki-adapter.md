# ADR-0005：Anki Add-on 保持为薄适配器

- 状态：已接受（设计）
- 关联：`SYNC-001`、`SYNC-002`、`SYNC-003`、`SYNC-004`

## 背景

Anki Add-on 运行在 Anki 自带 Python 环境，受版本、hook 和主线程约束。把领域规则和渲染复制到 Add-on 会造成两套实现，并增加升级风险。

## 决策

Add-on 只负责配对、连接、受管 Note 身份、Anki hooks 和 collection 操作。`vocabd` 持有领域规则和持久事件；同步采用幂等消费、ack、游标和重连对账实现最终一致。

## 代价

- 离线期间不能立即把所有 Anki 变化提交到服务。
- 必须处理重复事件、ack 丢失和身份漂移。
- 目标 Anki 版本的 hook 行为必须实测，而不能只依据静态设计。

## 重新评估条件

若 Anki 提供稳定的外部同步 API，或 Add-on 环境能够可靠共享同一领域包，可以重新划分逻辑；无论如何应避免出现两套独立业务规则。
