# ADR-0003：生成器使用原子文件系统 job

- 状态：已接受（设计）
- 关联：`IMPORT-001`、`SEC-003`

## 背景

外部生成器可能与主应用在不同时间运行，也不应导入主应用内部 Python 包。需要一种可检查、可重试、无需网络且能提供机器可读错误的边界。

## 决策

生成器在同卷 `staging` 中写完 job，再原子重命名到 `inbox`。`vocabd` 完整校验后，以整个 job 为数据库事务提交或拒绝，并把输入与结果归档到终态目录。

## 代价

- 需要定义目录生命周期、崩溃恢复和磁盘清理策略。
- 大 job 的完整校验会增加峰值内存或临时存储压力。
- 协议升级必须兼容已投递文件。

## 重新评估条件

若需要跨设备、远程生成、流式大批量导入或低延迟在线反馈，应比较队列或网络 API；单机异步生成仍优先使用文件协议。
