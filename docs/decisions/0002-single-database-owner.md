# ADR-0002：`vocabd` 是唯一数据库所有者

- 状态：已接受（设计）
- 关联：`DATA-001`、`CHANGE-001`、`API-001`

## 背景

管理客户端、预览器和 Add-on 都需要访问卡片。如果每个客户端直接操作 SQLite，业务规则、迁移、并发控制和 outbox 事务会分散到多个进程。

## 决策

只有 `vocabd` 可以打开业务数据库。其他组件通过版本化的本地 HTTP/WebSocket API 或离线导入协议交互。

## 代价

- API 客户端修改数据前必须确保服务可用并持有有效凭据。
- 本地单机产品仍需处理服务生命周期、认证和协议兼容。
- 简单查询多一次进程间通信。

## 重新评估条件

若服务生命周期成为不可接受的产品成本，或未来改用天然支持多客户端事务的存储服务，应重新评估边界；客户端直接共享 SQLite 不是默认退路。
