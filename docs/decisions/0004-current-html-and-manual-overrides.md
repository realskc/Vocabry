# ADR-0004：持久化当前 HTML，并允许 Anki 手工版本优先

- 状态：已接受（设计）
- 关联：`HTML-001`、`HTML-002`、`CARD-001`

## 背景

结构化字段适合自动生成和再渲染，但用户也可能直接在 Anki 中修改 `Front` / `Back`。若每次读取都从字段生成 HTML，手工修改会丢失；若把 HTML 变成唯一真源，又无法可靠恢复结构化含义。

## 决策

结构化字段是自动逻辑的输入真源，`front_html` / `back_html` 是持久化的当前交付物。Anki 回写可使手工 HTML 成为当前版本；之后修改结构化字段时，renderer 再次生成并覆盖当前 HTML。

## 代价

- 当前 HTML 可能与结构化字段不一致，必须记录 `html_origin`。
- 用户修改结构化字段会有意覆盖之前的手工 HTML。
- renderer 升级需要区分自动 HTML 和手工 HTML。

## 重新评估条件

若产品禁止 Anki 侧编辑，或引入可靠的 HTML 到结构化字段反向解析，可以简化双真相状态；在此之前必须显式保留来源和 revision。
