# ADR-0005：Anki Add-on 保持为薄适配器

- 状态：已接受、部分实现

## 背景

Add-on 运行在 Anki 自带 Python 与 Qt 环境，受版本、hook 和主线程约束。把领域规则和 renderer 复制到插件会形成两套实现并提高升级风险。

## 决策

Add-on 只负责配对、连接、受管 Note 身份、Anki hooks 和 collection 操作。`vocabd` 持有领域规则和持久事件；同步目标是至少一次投递、幂等应用、成功 ack 和断线后对账。

## 代价

- 服务离线时不能立即提交 Anki 变化，需要可靠的本地待办状态。
- 必须处理重复事件、ack 丢失、冲突和身份漂移。
- 目标 Anki 版本的 hook 与线程行为必须实测。

## 当前偏差

薄适配边界已经成立，但持久离线队列、全量对账和冲突恢复尚未实现。详见[同步设计](../../design/anki-sync.md)。

## 重新评估条件

若 Anki 提供稳定的外部同步 API，或插件环境能可靠共享领域包，可以重新划分逻辑；仍应避免两套独立业务规则。
