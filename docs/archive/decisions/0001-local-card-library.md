# ADR-0001：本地 Card 库独立于 Anki

- 状态：已接受、已实现

## 背景

卡片需要支持外部生成、结构化管理和完整历史。若把 Anki collection 当作唯一数据模型，生成与管理逻辑会被 Anki Note Type、运行状态和版本变化约束。

## 决策

Vocabry 维护独立的本地 Card 库。Anki 是第一版唯一复习下游，但不是结构化字段和版本历史的持有者。Anki 的人工卡面修改通过同步回到 Vocabry，成为当前 HTML 版本。

## 代价

- 必须处理 Card 与 Note 的身份映射和双向一致性。
- Anki 手工 HTML 与结构化字段可能有意不一致。
- 产品需要独立服务、数据库、备份和迁移。

## 重新评估条件

若产品确定永久只支持 Anki，且结构化管理、外部生成和独立历史都不再需要，可以重新比较直接使用 Anki collection 的简单性。
