# ADR-0004：持久化当前 HTML，并允许 Anki 手工版本优先

- 状态：已接受、已实现

## 背景

结构化字段适合自动生成和再渲染，但用户也可能直接在 Anki 中修改 `Front` / `Back`。每次读取都重新渲染会丢失手工修改；只保存 HTML 又无法可靠恢复结构化含义。

## 决策

结构化字段是自动逻辑的输入，`front_html` / `back_html` 是持久化的当前交付物。Anki 回写可使手工 HTML 成为当前版本；之后修改结构化字段时，renderer 再次生成并覆盖当前 HTML。单纯 renderer 升级不覆盖手工 HTML。

## 代价

- 当前 HTML 可能与结构化字段不一致，必须记录来源。
- 结构化字段修改会有意覆盖之前的手工 HTML。
- renderer 升级需要区分自动与手工 HTML。

## 重新评估条件

若产品禁止 Anki 侧编辑，或引入可靠的 HTML 到结构化字段反向解析，可以简化这套状态。
