# 版本更新归档

本目录按版本保存当时的目标、实施计划、取舍和验收记录，作用类似 [`archive/decisions/`](../decisions/README.md)：帮助未来维护者理解某一版本为何这样演进，但不作为当前产品或技术设计的事实源。

长期有效的用户语义必须收敛到 [`product/`](../../product/README.md)，跨模块技术约束必须收敛到 [`design/`](../../design/README.md)。当前完成度以 [`STATUS.md`](../../STATUS.md)、代码和测试为准。不能因为某项内容已经写入版本归档，就省略正式产品或设计文档中的记录。

- [v0.2.1：Word Query 词组查询](v0.2.1.md)

版本发布后保留对应归档，同时检查稳定的产品语义和跨模块设计已经分别进入 `product/` 与 `design/`，并更新 `STATUS.md`。过时实现步骤可以保留为历史，但归档页必须显式链接当前事实源，避免被误当成现行规范。
